# 🍪 유리네과자집 예약 기록 달력(reservation-calendar)

## 💡 프로젝트 개요

이 프로젝트는 제 여자친구가 실제로 운영중인 카페의 예약 정보를 달력에 표시하여 픽업시간, 예약자, 메뉴, 연락처, 결제 완료 여부, 픽업 완료 여부를 확인할 수 있는 시스템입니다.

관리자는 제작자로부터 아이디와 패스워드를 받아 예약 현황을 달력 형태로 확인할 수 있으며, 예약을 추가/수정/삭제 할 수 있습니다.

---

React, Spring Boot, PostgreSQL을 Docker 기반으로 통합하여 개발, 배포 환경을 일원화한 예약 관리 시스템입니다.

## 💡 기능 목록 (v1)

### 1️⃣ 예약 생성

- 예약자, 메뉴, 연락처, 예약금액을 입력받아 예약 생성
  - 해당 값이 누락됐을 경우 예약이 생성되지 않는다.
  - 각 입력 필드의 입력 형식이 맞지 않을 경우 예약이 생성되지 않는다.
  - 예약 날짜가 현재보다 과거일 경우 예약이 생성되지 않는다.

### 2️⃣ 예약 조회

- 특정 날짜 혹은 달 단위로 예약 목록 조회
  - 픽업시간을 지정하지 않을 경우 '시간 정보 없음' 이라고 출력된다.

### 3️⃣ 예약 수정

- 예약 정보(메뉴, 결제 상태 등) 수정
  - 존재하지 않는 예약 ID로 수정 요청한 경우 예외가 발생한다.
  - 각 입력 필드에 유효하지않은 값이 입력되면 예외가 발생한다.
  - 과거의 예약을 변경할 상황이 생길 수 있기에 시간을 과거로 수정 가능하다.

### 4️⃣ 예약 삭제

- 잘못된 예약 혹은 취소 요청 삭제
  - 존재하지 않는 예약 ID로 삭제 요청하면 예외가 발생한다.

### 5️⃣ 예약 출력

- 예약이 있는 날짜에 표시
- 해당 날짜를 들어가면 예약 상세확인

### 6️⃣ 로그인, 로그아웃

- 할당받지 않은 유저정보를 입력하면 예외가 발생한다.
- 로그인 시 액세스 토큰과 리프레시 토큰이 발급된다.
- 리프레시 토큰은 DB에 저장된다.
- 로그아웃 시 DB에 저장한 리프레시 토큰이 삭제된다.
- 액세스 토큰이 만료되면 리프레시 토큰을 확인하여 유효하다면 액세스 토큰을 재발급한다.
